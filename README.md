# Обнаружение аномальной активности респондентов по брендам

Алгоритм для поиска аномально высокой активности пользователей по отдельным брендам. Результат используется для очистки выборки перед анализом популярности брендов.

## Логика работы

1. **Агрегация данных**  
   Для каждого респондента, дня, категории и бренда вычисляется `daily_ots = Weight * количество_запросов` (строго по ТЗ).

2. **Пороги**  
   - Глобальный порог OTS: 99.9% перцентиль всех значений `daily_ots`  
   - Порог пика бренд-дня: 99.9% перцентиль суммарного OTS бренда за день

3. **Правила аномалии**  
   Респондент считается аномальным по бренду в день, если:  
   - `daily_ots >= глобальный_порог` И  
   - (`доля респондента в OTS бренда за день >= 0.5` ИЛИ `суммарный OTS бренда за день >= порог_пика`)

4. **Удаление**  
   Если респондент признан аномальным хотя бы по одному бренду в день, удаляются **все его строки за этот день** (по всем брендам и категориям).

## Запуск

1. **Установите зависимости**  
   `pip install -r requirements.txt`

2. **Укажите путь к данным**  
   Откройте файл `solution_AbramovaAO_ШЦТ-111.ipynb` и в начале кода измените переменную `DATA_DIR` на путь к вашей папке с данными (например, `data_train/month=2025-06-01`).
   Код изучает только один месяц.

4. **Запустите скрипт**  
   Выполните все ячейки ноутбука.

5. **Результаты**  
   - `output/anomalies.csv` – список уникальных пар `SubjectID, researchdate` для удаления  
   - `output/anomaly_reasons.csv` – причины аномалий с детальным объяснением  
   - `output/plots/total_ots_before_after.png` – динамика общего OTS по дням  
   - `output/plots/category_ots_change.png` – процент удалённого OTS по категориям  
   - `output/plots/daily_anomaly_count.png` – количество аномальных респондентов по дням

## Параметры (регулируются в коде)

| Параметр | Значение | Описание |
|----------|----------|----------|
| SHARE_THRESHOLD | 0.5 | Доля респондента в OTS бренда за день |
| OTS_QUANTILE | 0.999 | Глобальный порог daily_ots (99.9%) |
| PEAK_QUANTILE | 0.999 | Порог пика бренд-дня (99.9%) |

## Почему такие пороги?

**OTS_QUANTILE = 0.999 (99.9% перцентиль)**

Берём только топ-0.1% самых больших значений daily_ots. Почему? Если взять 99% — будет слишком много аномалий (1% всех данных). Если взять 99.99% — найдём всего пару выбросов, но пропустим остальных. 99.9% — золотая середина.

**PEAK_QUANTILE = 0.999 (99.9% перцентиль)**

То же самое, но для суммарного OTS бренда за день. Ловим дни, когда бренд получил аномально много запросов.

**SHARE_THRESHOLD = 0.5 (доля 50%)**

Если один человек сделал больше половины всех запросов по бренду за день — это подозрительно. 50% — точка, где респондент начинает доминировать над всеми остальными вместе.

**Учёт редких брендов**

Защита от ложных срабатываний в редких брендах обеспечивается глобальным порогом daily_ots (99.9% перцентиль). Даже если доля респондента 100%, он не попадёт в аномалии, если его daily_ots меньше глобального порога. Это автоматически отсекает бренды с малым количеством запросов.
